Interactive Charts

Visualizations are often static, with the purpose of presentation rather than the exploration of data (Malik and Ünlü 2011). To many people, a graph will be made in Excel, saved as an image and then inserted into a report. By not allowing the user to interact with the data, it limits interaction and exploration with the results.

Most users of R will be familiar of the package ggplot2 (Wickham and Chang 2018), which is frequently used as the main plotting system in R. The plotly (Sievert et al. 2017) package builds upon the ggplot2 and provides a tool to easily convert graphs into an interactive, web-based version.

Figure 1 shows a basic interactive Shiny plot. The user can hover over bars to find out more details about the group, and also use the legend to turn on and off layers.

# Code to build basic ploty graph
library(ggplot2)
library(plotly)
p <- ggplot(data = diamonds, aes(x = cut, fill = clarity)) +
            geom_bar(position = "dodge")
ggplotly(p)

Figure 1: Interactive Shiny Plot

Plotly is recommended as many R users will be familiar with ggplot. However, there are lots of other options available for plotting graphs and graphics, and many build off existing Javascript libraries. These include:

  • dygraphs for R: an R interface to the dygraphs JavaScript charting library.
  • scatterD3: R scatter plot htmlwidget based on D3.js
  • rbokeh: provides an R Interface for Bokeh

There are many more examples provided in the htmlwidgets gallery.

Interactive Maps

Static maps can be quite challenging to make effectively. It is difficult to know which scale at which they should be drawn. If you produce a national map, most users will want to zoom into their area and find out more results about their local area.

Maps can easily be made using the leaflet (Cheng, Karambelkar, and Xie 2017) package. Using the following example taken from here, we can create maps in only a few lines of code:

library(leaflet)
m <- leaflet() %>%
  addTiles() %>%  # Add default OpenStreetMap map tiles
  addMarkers(lng=-1.39595, lat=50.93463, popup="Southampton University")
m

Flowcharts and Visualisations

It can often be useful to visualise flowcharts or networks in R. I particularly like to use them when explaining my data analysis process. Figure 3 highlights a basic structure for a flowchart, using the package DiagrammeR (Iannone 2018) t build a GraphViz plot:

library(DiagrammeR)
grViz("digraph a_nice_graph {

        graph [layout = dot, rankdir = LR]
        
        # node definitions with substituted label text
        node [fontname = Helvetica, shape = rectangle, style = filled, fillcolor = WhiteSmoke]        
        rec1 [label = '@@1']
        rec2 [label = '@@2']
        rec3 [label = '@@5']
        rec4 [label = '@@6']

        node [fontname = Helvetica, shape = oval]
        ova1 [label = '@@3']
        ova2 [label = '@@4']

        # edge definitions with the node IDs
        rec1 -> ova1 -> rec2 -> ova2 -> rec3
        ova2 -> rec4
        }
      
        [1]: 'raw data'
        [2]: 'cleaned data'
        [3]: 'clean and tidy'
        [4]: 'describe'
        [5]: 'report'
        [5]: 'website'
      ")

Figure 3: An example Flowchart from DiagrammeR
